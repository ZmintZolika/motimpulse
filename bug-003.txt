Azonosító: BUG-003

Összefoglalás: Hiányzó jelszókövetelmény-tájékoztatás és hibás API válasz (302 Redirect) validációs hiba esetén

Súlyosság: Magas

Prioritás: Magas

Státusz: Javítva

Komponens: Frontend (Regisztráció) & API Konfiguráció

Környezet: Helyi fejlesztői környezet (Localhost) / Böngésző

Előfeltételek: A regisztrációs oldal (/register) megnyitása.

Lépések:
1. A regisztrációs űrlapon a jelszó mező ellenőrzése (segédszöveg keresése).
2. Regisztráció megkísérlése rövid (pl. 3 karakteres) jelszóval.
3. A Fejlesztői Eszközök (F12) Network fülén a válasz vizsgálata.

Elvárt működés:
1. A felületen előzetesen szerepeljen: "Min. 8 karakter".
2. Hibás kitöltéskor az API 422 Unprocessable Entity (JSON) választ küldjön a konkrét hibaüzenettel.
3. A Frontend jelenítse meg a kapott hibaüzenetet: "A jelszó túl rövid".

Tényleges működés:
1. Nincs előzetes tájékoztatás a jelszó hosszáról.
2. Validációs hiba esetén a szerver 302 Found (Redirect) választ küld JSON helyett.
3. A Frontend emiatt nem tudja feldolgozni a választ, és csak egy általános "Hiba történt a regisztráció során" üzenetet jelenít meg.

Oka:
A Frontend kéréséből hiányzik az 'Accept': 'application/json' fejléc, ezért a Laravel HTML választ (átirányítást) küld.

Javaslat:
1. UI: Szöveges tájékoztatás pótlása a mező alatt.
2. Kód: A fetch hívás headerjének kiegészítése az 'Accept': 'application/json' sorral.

Felelős: Nincs hozzárendelve
